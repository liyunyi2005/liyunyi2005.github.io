嵌入式项目周期长、角色多，文档是“协作的接口”。下面是一个轻量但可执行的文档体系。

## 1. 核心文档类型

- 需求文档：目标、范围、约束、优先级
- 设计文档：硬件选型、架构图、功耗预算
- 接口文档：协议、数据格式、时序
- 测试文档：测试计划、记录、缺陷跟踪
- 复盘文档：问题根因、改进清单

## 2. 最小模板示例

```
接口名称：串口上位机通信
波特率：115200
帧格式：AA 55 LEN CMD DATA CRC
超时：50ms
重传：3次
```

## 3. 落地建议

- 标题统一规则便于搜索
- 文档内保留变更记录
- 关键指标尽量量化

文档不是“负担”，而是工程质量的保险。
